# E-030 Workflow Invoke Outcome-Pipeline Exactly-Once (#763)

## Product Promise

#763 restructured `_core.workflow_wrapper` to
`.wrap(get_wf_invoke).intercept(check_and_init)`. `check_and_init` decides
`should_execute`; on the not-execute branch it must return the recorded
result (or re-raise the recorded error) without running the body, populating a
shared-closure `init_status` only on the execute branch. For async this splits
across `asyncio.to_thread` threads. The invariant: body runs exactly once;
concurrent and replay callers get the recorded outcome; a recorded ERROR
re-raises the original exception type, never a pipeline internal.

## Fault Dimensions

### concurrent-same-id-and-completed-replay

Purpose: falsify exactly-once/recorded-outcome under the new pipeline for
concurrent same-id and completed-async re-invocation.

Trigger: (a) start two concurrent invocations of one workflow id (async and/or
via SetWorkflowID); loser hits `should_execute=False` and must await/return the
winner's recorded result. (b) await an async workflow to completion, then
re-invoke the same id — must return the recorded result. (c) an async workflow
whose body raised: re-invoke must re-raise the ORIGINAL exception type.

Reachability evidence: `dbos/_core.py workflow_wrapper` (`.wrap(get_wf_invoke)
.intercept(check_and_init)`); async path `dbos/_outcome.py Pending._intercept`
/`_wrap` run on separate `asyncio.to_thread` threads.

Expected product behavior: body side-effect fires exactly once across all
invocations; every caller returns the identical recorded result; a recorded
error re-raises its original type (not `KeyError`/`NoResult`/pipeline leak).

Oracle impact: FAIL on a second body execution, divergent results between
callers, a lost result, or a recorded error surfacing as a pipeline-internal
exception type.

Replay: `.workers/run-with-postgres.sh .workers/python-runtime.sh .workers/workloads/workflow-invoke-outcome-pipeline/workflow_invoke_outcome_pipeline_workload.py --rung rung-001-concurrent-and-replay-invoke --case <case-id>`.
