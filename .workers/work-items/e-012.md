## Fault Details

### live-stream-resume-listener-offsets

Purpose: prove that live DBOS stream readers can resume from explicit offsets,
wake through LISTEN/NOTIFY or modeled fallback polling, terminate when the
writer workflow completes without an explicit close, and clean listener state.

Trigger point: readers consume a prefix from a live writer, disconnect or block
at the live end, then resume from a recorded offset while writes or terminal
workflow transitions continue.

Injection: vary runtime sync/async, client sync/async, listener-disabled
fallback, unclosed stream termination, and post-relaunch offset reads. Use long
polling intervals where needed so sub-interval delivery proves notification
wakeup rather than accidental fixed sleeps.

Reachability evidence: cloud run `01KVVR752868AG2HW0ZWJ5MKTA` reached all
three modeled cases and passed 48/48 invariants.

Expected product behavior: readers see exactly `modeled_sequence[offset:]` with
no prefix replay, gaps, duplicates, or post-close values; blocked readers exit
after terminal writer status; all listener payloads are unregistered after
normal close, fallback, termination, and relaunch paths.

Oracle impact: any suffix mismatch, stale `streams_map` entry, missed terminal
exit, or runtime/client disagreement violates the communication-state ledger.

## Oracle

The oracle is an independent stream/listener ledger keyed by workflow ID,
stream key, reader path, requested offset, and expected terminal behavior. It
checks:

- observed suffix equals the modeled stream sequence from the requested offset;
- runtime sync, runtime async, client sync, and client async observations agree
  where included;
- live in-process readers wake inside the recorded latency bound with polling
  interval evidence;
- disabled-listener/client fallback readers finish inside fallback bounds;
- blocked readers at an unclosed live end exit after workflow terminal state;
- `streams_map` is empty for modeled listener payloads after every reader path;
- post-relaunch reads resume from the requested offset without resurrecting
  stale listeners or mutating terminal writer state.
