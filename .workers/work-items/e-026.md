## Fault Details

### cancel-children-terminal-immutability

Purpose: verify the cancellation semantics added by PR `#701` and PR `#703` beyond the narrow product tests.

Fault model: parent workflows can have running children, grandchildren, and queued descendants while an operator or client cancels the parent. A regression can cancel too little, cancel unrelated rows, leave queued descendants visible/dequeueable, let a late body return overwrite terminal `CANCELLED`, duplicate durable step output on resume, or disagree between runtime and `DBOSClient` observation paths.

Oracle impact: child graph, public status rows, queued-workflow listings, handle result exceptions, step counters, and runtime/client observations must agree with the independent lifecycle graph model after every command.
