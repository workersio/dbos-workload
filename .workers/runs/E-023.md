## Classification

Product finding candidate for the claimed rung.

The failure signature says the SQLite locked-database/table retry path records
the workflow as terminal `ERROR` before the datasource transaction body can
retry after the external writer lock is released. That violates the rung
promise for retryable DBAPI concurrency failures: retryable lock failures
should be transient, should not be recorded as terminal datasource errors, and
should eventually produce one durable success plus replay without duplicate app
side effects.

The Postgres serialization, Postgres async deadlock, and async non-retryable
control cases passed in the same cloud matrix, so the recorded candidate is
specific to the SQLite locked datasource retry path.
