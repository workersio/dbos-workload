## Classification

Product finding candidate for the claimed rung.

The target delivers sync and async client event values promptly enough in the
modeled no-listener client path, and event update reads preserve current values.
However, when the target workflow completes successfully without setting the
requested event key, `DBOSClient.get_event` waits close to the full client
timeout before returning `None` instead of concluding the modeled terminal miss
promptly. This violates the E-025 promise that client event waits return stable
`None` for modeled terminal misses within the prompt polling/terminal bound.

## Review

Verdict: `KEEP`

The workload adds a client-facing no-listener `get_event` surface not covered
by the existing runtime fallback, cancellation, stream, or replay rungs. It
would catch regressions where a client waits until timeout despite event writes,
blocks the asyncio loop, returns stale event values after updates, leaks waiter
map entries, or misses terminal workflow state. The final evidence isolates the
active failure to terminal missing-event liveness while green controls preserve
event delivery and update behavior.
