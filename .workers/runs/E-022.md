## Classification

Product finding candidate for the claimed rung.

The failure signature says many concurrent DBOS warm starts against an already
current Postgres system schema still block behind the migration advisory lock.
That violates the protected liveness promise from PR `#677`: an up-to-date
schema should let startup skip lock acquisition and return while an unrelated
session still owns the migration lock.

The stale and partial schema safety side of the rung passed before the failing
case, so the recorded candidate is specifically the rolling-deploy warm-start
liveness path, not a false early exit or repair failure.
