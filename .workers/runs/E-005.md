## Classification

Finding.

The workload created workflow A under `wio_schema_a_7420` and workflow B under
`wio_schema_b_7420`, then constructed `DBOSClient` A followed by `DBOSClient` B
in one process. Independent quoted physical SQL proved the rows were placed in
their expected schemas. After client B initialized, public reads through client
A returned B's workflow and missed A's workflow.

The target code path supports the mechanism: `SystemDatabase.__init__` calls
`SystemSchema.set_schema(self.schema)`, and `SystemSchema.set_schema(...)`
mutates class-level SQLAlchemy table `.schema` fields shared by every system
database object in the process. `DBOSClient` stores its `SystemDatabase`, but
later client construction mutates the shared table metadata used by earlier
client methods.
